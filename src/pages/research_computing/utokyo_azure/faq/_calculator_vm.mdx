Microsoft Azure が用意する[**料金計算ツール**](https://azure.microsoft.com/ja-jp/pricing/calculator/)を利用して見積もります．
- この例は以下の２つの構成の計算例です
  - [**Virtual Machine**](/research_computing/utokyo_azure/virtualmachine/): 仮想マシン（OS用ストレージ含む）
  - [**Azure Blob Stoarge**](/research_computing/utokyo_azure/storage/): データ用ストレージ

## 1. 仮想マシン

- 製品から `コンピューティング` > `Virtual Machine` を選択してください．
- 画面下の**見積もり**に`Virtual Machine`が追加されます．

#### インスタンス

| **項目** | **選択** |
| -------- | ------- |
| リージョン | Japan East |
| オペレーティングシステム | Linux |
| タイプ | ubuntu |
| レベル | Standard |
| カテゴリ | High Performance Compute |
| インスタンスシリーズ | H-series |
| インスタンス | 16CPU,112G RAM,2T [一時ストレージ](https://learn.microsoft.com/ja-jp/azure/virtual-machines/managed-disks-overview#disk-roles) |

以上一式を一カ月分(730時間)
- UTokyo Azure では課金は**従量課金**のみです．節約プラン，予約プランは選択できません．
- 常時立ち上げ状態を想定しています．定期的にシャットダウン及び起動を繰り返して節約する場合は，起動している予定の時間だけ入力してください．

#### マネージドストレージ
OS用に30Gのマネージドストレージを追加

| **項目** | **選択** |
| -------- | ------- |
| レベル | Standard HDD |
| サイズ | S4(32GB) |

以上一式を１か月分

- **ストレージトランザクション**，**帯域幅**
  - 通常は見積もらなくて問題ありませんが，リージョン間に跨ぐ通信が発生する場合は通信料が発生しますので，その場合は帯域幅に条件を入力してください．できる限り同一リージョン内で構築ください．
- **サポート**：Basic のままで構いません．

### 小計 **約 1,675ドル** ([参考](https://azure.com/e/2cc3b709e1114ebdb00288b099b8db5c))

## 2. データ用ストレージ

- 製品から `ストレージ` > `ストレージアカウント` を選択してください．
  - Azure Files の場合は `Azure Files` を選択してください．必要な入力項目も変わりますので，要求された項目を入力して対応ください．
- 画面下の**見積もり**に`ストレージアカウント`が追加されます．

#### ストレージ仕様

| **項目** | **選択** |
| -------- | ------- |
| リージョン | Japan East |
| タイプ | ブロック Blob Storage |
| パフォーマンス | 標準 |
| ストレージアカウントの種類 | Blob ストレージ |
| ファイル構造 | フラットな名前空間 |
| アクセス層 | ホット |
| 冗長性 | LRS |

- **容量**：100TB
- 課金は UTokyo Azure では**従量課金のみ**です．予約プランは選択できません．
- **操作とデータ転送**：初期値のままで計算しています．ご自身の利用予定に合わせて更新してください．

### 小計 **約 2,000ドル** ([参考](https://azure.com/e/3b7371e33f704716b31df9d0dc5018c0))

## 計算結果

仮想マシン: **1,675 ドル** + データ用ストレージ: **2,000 ドル** = **3,675 ドル** / 月額

- 実際は利用状況や細かい条件で増減するので，おおよその目安とお考え下さい．仮想マシンは使っていない時間をシャットダウンさせておくなどで利用料を減らすことができます．（ストレージは容量で固定です）
