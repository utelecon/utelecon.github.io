## [料金計算ツール](https://azure.microsoft.com/ja-jp/pricing/calculator/)

Microsoft Azure では料金を事前に見積もるためのツールが用意されています．一定額までは寄付金で賄われるとは言え，例えばAIサービスの利用で高機能な仮想マシンを利用するとあっという間に上限に達し不足分を予算から支出することになってしまいますので，自分がデプロイしたサービスがどのくらいの利用料で使えるのか事前に計算し，想定を超えるようであればこまめにサービスを停止させて課金を止めるなどの対応をしてください．

## 計算例

仮想マシンは選択したCPU/GPUの数やストレージ量などの資源で計算されます。

- `コンピューティング` > `Virtual Machine` を選択します．
- 画面下の**見積もり**に Virtual Machine が追加されるので，ここでデプロイする予定の仮想マシン仕様を入力します。
  - リージョンやカテゴリ，タイプなどで選択できるインスタンスが変わってきます．ちょうどいい選択肢が出ない場合はリージョン等の選択肢を変えてみてください．

#### インスタンス

| **項目** | **選択** |
| -------- | ------- |
| リージョン | Japan East |
| オペレーティングシステム | Linux |
| タイプ | ubuntu |
| レベル | Standard |
| カテゴリ | GPU |
| インスタンスシリーズ | NC A100 v4-series |
| インスタンス | 24GPU,220G RAM,1T 一時ストレージ |

以上一式を一カ月分(730時間)
- 節約プラン，予約プランは選択できません．

#### マネージドストレージ
データ用に1TBのマネージドストレージを追加

| **項目** | **選択** |
| -------- | ------- |
| レベル | Standard HDD |
| サイズ | S4(1024GB) |

以上一式を１か月分

#### ストレージトランザクション，帯域幅

通常は見積もらなくて問題ありませんが，リージョン間に跨ぐ通信が発生する場合は通信料が発生しますので，その場合は帯域幅に条件を入力してください．できる限り同一リージョン内で構築ください．

#### サポート

Basic のままで構いません．

## [計算結果](https://azure.com/e/57b0185c416a422b88fbb14a65ef3d60)

上記例では月額約 4,000ドルとなりました．実際は利用状況や細かい条件で増減するので，おおよその目安とお考え下さい．CPU/GPUは使っていない時間をシャットダウンさせておくなどで利用料を減らすことができます．
